누가 사용하는 서비스인가요?

사용자 범위에 따라 로그인·요금·구조가 달라집니다

A

나 혼자 (개인용)

로컬 또는 사설 서버에서 혼자 사용. 로그인 불필요, API 키는 내 것 하나만.

장점

  • 구현 단순
  • 인증 불필요
  • 비용 예측 쉬움

단점

  • 타인과 공유 어려움
B

팀·사내 (소수 인원)

회사나 팀 내부용. 간단한 인증(비밀번호 또는 구글 로그인), API 키는 공용.

장점

  • 팀원과 공유
  • 간단한 접근 제어

단점

  • 인증 기능 필요
  • API 비용 공유 관리
C

공개 서비스 (누구나)

불특정 다수가 사용하는 SaaS. 회원가입, 사용량 제한, 요금제 등 필요.

장점

  • 수익화 가능
  • 광범위한 사용자

단점

  • 구현 복잡도 높음
  • 보안·인프라 관리 부담
