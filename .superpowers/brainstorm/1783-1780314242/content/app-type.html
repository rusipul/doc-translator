어떤 형태의 앱으로 만들까요?

파일을 번역하는 툴의 기본 형태를 결정합니다

A

웹 앱

브라우저에서 파일을 업로드하고 번역 결과를 다운로드. 설치 없이 어디서든 사용 가능.

장점

  • 설치 불필요
  • 어디서든 접근
  • 공유 쉬움

단점

  • 서버 호스팅 필요
  • 대용량 파일 업로드 제한
B

데스크탑 앱

내 컴퓨터에 설치해서 사용. 드래그앤드롭으로 파일 처리, 오프라인 일부 지원.

장점

  • 빠른 처리
  • 파일 크기 제한 없음
  • 로컬 보안

단점

  • 설치 필요
  • OS별 빌드 필요
C

CLI 도구

터미널에서 명령어로 실행. 자동화, 배치 처리에 최적화.

장점

  • 자동화 가능
  • 스크립트 통합
  • 개발 간단

단점

  • 비개발자 사용 어려움
  • GUI 없음
